Identical to |JS::Compile| for UTF-8, except this function directly parses its UTF-8 input without inflating it to UTF-16 and parsing that.

The "DontInflate" suffix and (semantically unobservable) don't-inflate characteristic are temporary while bugs in UTF-8 compilation are ironed out. In the long term |JS::Compile| for UTF-8 will just never inflate, and this separate function will die.

NOTE: UTF-8 compilation is currently experimental, and it's possible it has as-yet-undiscovered bugs that the UTF-16 compilation functions do not have. Use only if you're willing to take a risk!
